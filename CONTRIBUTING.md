# Contributing

## About this playbook

This playbook was created by [Martin Dean](https://github.com/Asborien) and is maintained as part of the [Certified Coach](https://www.certified-coach.com) project. The practices documented here are extracted from daily production use — they're opinionated because they've been tested.

## Can I contribute?

Yes, but selectively. This isn't a community framework where all ideas are welcome. Every practice in this playbook must be:

1. **Tested in production** — not theoretical. You've used it on a real project.
2. **Broadly applicable** — works across tech stacks, not specific to one framework.
3. **Proven to solve a real problem** — with a clear before/after.

## How to contribute

1. **Open an issue first.** Describe the practice, the problem it solves, and your experience using it. Don't submit a PR without discussion.
2. **Keep it concise.** Each practice doc should be readable in 5 minutes.
3. **Include the "why".** Every practice needs a rationale, not just a rule.

## What we're not looking for

- Framework-specific patterns (this is stack-agnostic)
- Untested ideas ("I think this would work")
- Complex tooling (the playbook should work with Claude Code and git — nothing else required)
- Rewrites of existing practices (refine, don't replace)

## Code of conduct

Be professional. Be constructive. Be specific. This isn't a debate forum — it's a working playbook.
